CSES Sorting and Searching Solution

Sorting and Searching题解

CSES题目集：https://cses.fi/problemset/

Sorting and Searching

Distinct Numbers

扔进 set 里面维护即可

Apartments

对 a,b 数组分别排序，然后双指针跑一遍

Ferris Wheel

排序，然后双指针跑一遍

Concert Tickets

把所有数的负数扔进 multiset，然后对于每一个 xxx，使用 set.lower_bound(-x) 去找，特别判断容器是否为空

Restaurant Customers

把 l,r 离散化一下，然后用前缀和去求

Movie Festival

把电影按照结束时间排序，然后遍历一遍就可以得到答案

Sum of Two Values

用 map 存每个数出现的位置，然后遍历每个数 aia_iai​ ，看 map 里面有没有 x−aix-a_ix−ai​ ,记下两个不同的下标即可

Maximum Subarray Sum

定义 sum=0 ，遍历一遍数组，sum+=a[i]，如果 sum<0，sum=0 。答案就等于此过程中 sum 的最大值

Stick Lengths

找中位数即可

Missing Coin Sum

先排序，然后定义 res=0，遍历数组，如果 a[i]<=res+1，也就是说 1 到 res+a[i] 的数都可以凑到，否则答案就是 res+1

Collecting Numbers

用 set 模拟即可

Collecting Numbers II

可以看出如果 id[x]<id[x−1]id[x]<id[x-1]id[x]<id[x−1] 说明轮数就要加一，因此可以先记录每个数出现的下标，同时算出在交换之前的轮数，交换时判断 id[x−1],id[x],id[x+1],id[y−1],id[y],id[y−1]id[x-1],id[x],id[x+1],id[y-1],id[y],id[y-1]id[x−1],id[x],id[x+1],id[y−1],id[y],id[y−1] 这些的关系，同时更新答案即可

Playlist

map 存每个数的出现次数，然后跑双指针

Towers

用 multiset 处理，如果一个数大于等于集合里面的最大值，就直接插入，否则通过 upper_bound 删除大于这个数的最小值，然后再插入。最后答案就是集合的元素个数

Traffic Lights

使用两个 multiset s,t 。s 存哪些点是红绿灯，t 存目前最长的路。一开始 s 存 0 和 n 。t 存 n 。每次输入一个数， 设 p=s.lower_bound(x)，q=p-1 , 代表受影响的段。然后在 t 中删除 *p-*q ,然后更新新的段。t 中的最大值就是答案

Josephus Problem I && Josephus Problem II

直接用 pbds 里面的数据结构，暴力模拟即可，时间复杂度为 O(nlog n)O(nlog\ n)O(nlog n)

Nested Ranges Check && Count

分两步去求，先求每个区间包含多少区间，对于区间 [l,r][l,r][l,r] 按照 l 排序，这样在第 i 个区间覆盖之前的区间的左边界，只要找到它之前的所有右区间比它小的数量就是答案，被包含反之即可

Room Allocation

先按照到达时间排序，再创建一个优先队列，设置为离开时间越早优先级越高，遍历排完序之后的数组，如果队列中存在离开时间比其到达时间早的，则可以入住，否则需要额外开房间

Factory Machines

二分答案

Tasks and Deadlines

按照持续时间从小到大排序即可

Reading Books

定义 max 为读完某一本书最长的时间（如果有多个，也一样），sum 为一个人读完的总时间。可以假设1号先读 ，如果 max>=sum-max，意味着当1号读完 max 之前，2号读完其他书之后，只能干等着1号读完，那么此时答案就是 2*max。反之， 当1号读完 ，为了避免冲突，他应该先读2号读过的书，最后2号读完所有书的时间为 sum ，而1号的时间为 max+sum-max，那么此时答案就是 sum

Sum of Three Values

用 map 存每个数出现的下标，然后双重循环找前两个数，用 map 找最后一个数

Sum of Four Values

用 map 存每两个数相加出现的数，以及它们的下标，然后循环找其中一个数，用 map 找另一个数

Nearest Smaller Values

单调栈，若某个数大于等于栈顶元素，则直接输出栈顶元素的下标，并入栈。否则，不断出栈，真到出现小于或者栈为空，输出下标，并入栈

Subarray Sums I

双指针，定义 i,ji,ji,j 两个指针，如果 i,ji,ji,j 这段区间和大于 x ,则 i++ , 否则 j++

Subarray Distinct Values

双指针，两个指针之间代表满足的子数组，尾指针不断向后扩展，如果不满足条件，头指针就往前扩展，同时记录答案即可

Array Division

二分答案，然后判断是否可以划分成 kkk 组即可

Sliding Median

依然可以用pbds

Sliding Cost

易得最小花费就是把所有数变成中位数，类似于滑动窗口，假设一个已经算好结果的窗口答案为 ansians_iansi​ ，那么滑动到下一个窗口，如果中位数没有变化，那么答案就是 ansi−abs(a[i−k]−mid)+abs(a[i+1]−mid)ans_i-abs(a[i-k]-mid)+abs(a[i+1]-mid)ansi​−abs(a[i−k]−mid)+abs(a[i+1]−mid) ，如果中位数变化了，我们分奇偶讨论，如果是奇数，比中位数大的和比中位数小的数是一一对应的，即使中位数发生变化，也没有影响，那么答案就是上式，如果是偶数，相比奇数会多出一个数，那么这个数需要单独处理，也就是说它会产生额外贡献，而它的贡献就是中位数变化的绝对值

Movie Festival II

依然将电影时间按照结束时间排序，把所有人的看完电影的时间放进 set 里面，对于每一部电影，找到最接近且小于等于该部电影开始时间的人，修改他的看完电影时间即可
