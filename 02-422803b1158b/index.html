Hemose on the Tree

题目链接：https://codeforces.com/contest/1670/problem/E

题目大意：给你 n 个节点的树，把 [1,2n−1][1,2n-1][1,2n−1] 的权值分配给树上的节点和边。然后你可以选择任意一个点当根，要求根节点到任意点或边的最短路径异或和的最大值最小

解题思路：首先异或和的最小值不小于 n，因为要满足小于 n ，那么根节点的权值就得小于 n ,然后其它的节点和边的最高位都不能为 1，而这显然是不可能的。我们把剩下来的权值处理一下，让 x,x+n{x,x+n}x,x+n 凑成一对，这样它们异或就是 n ，总共又有偶数个，现在只要分配得当就可以得到答案的最小值 n 。让根节点的权值就为 n ,它到每个节点的异或和要么为 0，要么为 n 。如果当前走到的子节点的异或值为 0 ,那么接下来的边权就要小于 n ，否则大于 n，这样构造即可。

Tokitsukaze and Two Colorful Tapes

题目链接：https://codeforces.com/contest/1678/problem/E

题目大意：给你两个数组 a,b，两个数组都是 1-n 的排列，你可以给它们赋值，也要满足是一个全排列，问 ∑abs(val[a[i]]−val[b[i]])\sum abs(val[a[i]]-val[b[i]])∑abs(val[a[i]]−val[b[i]]) 最大是多少

解题思路：两个数组之间的关系可以用环组成，如果我们给环上每个点赋上最大值，最小值，次最大值，次最小值…，这样结果就是最大的，偶环这样是没问题的，而奇环的前 n-1 个这样构造，最后一个数，不管填什么对于结果的贡献都是不变的，因此奇环的最后一个应该填中间的，所以我们可以先把可以填的填完，最后所有奇环的最后一个在剩下的随便填就可以满足答案的最优性了

Nearest Leaf

题目链接：https://codeforces.com/contest/1110/problem/F

题目大意：给你一棵树，树的节点编号是通过欧拉序得到的，已知节点的连接关系和边权，多次查询，问以 uuu 为起点到编号为 [l,r][l,r][l,r] 之间的一个任意叶子节点的距离最短是多少，确保 [l,r][l,r][l,r] 里面至少有一个叶子

解题思路：可以离线查询，也就是说对于每一个节点 uuu 查询它所有的询问。查询的答案就是 dep[u]+min(dep[v]−2∗dep[LCA])dep[u]+min(dep[v]-2*dep[LCA])dep[u]+min(dep[v]−2∗dep[LCA]) ，当某个节点成为 LCALCALCA 时，把它所有子节点的深度都减去 2∗dep[u]2*dep[u]2∗dep[u] ，这样就可以通过 dep[u]+∑lrmin(dep[i])dep[u]+\sum_{l}^{r}min(dep[i])dep[u]+∑lr​min(dep[i]) 得到答案，用线段树维护即可
